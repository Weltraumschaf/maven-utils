Basis POMs fuer Sourcecode Export in Export-Verzeichnis (./export) legen.
  - de.icongmbh.commons-2.0.0.pom
  - de.icongmbh.dope.connect-1.2.0.pom
  - de.icongmbh.dope.dobre-1.2.6.pom

Sourcecode exportieren fuer jedes POM.
  mvn -f <POM file name> scm:export -DbaseDir=<export directory> -DexportDirectory=<POM file name> -N
  $ find ./export -mindepth 1 -maxdepth 1 -name \*.pom -exec mvn -f {} scm:export -DbaseDir=./export -DexportDirectory={} -N \;
  Sollte es Schwierigkeiten beim Export geben, die Sourcen manuell bereitstellen:
  mkdir -p ./export/<POM file name>
  SVN:
    svn export -r <scm TAG element in POM file> <scm URL without 'scm:svn:' in POM file> ./export/<POM file name> 
  GIT:
    git archive --format=tar --remote=<scm URL in POM file without 'scm:git:'> <scm TAG element in POM file> [<project directory in repository>/ ...] | tar -x -C ./export/<POM file name>/

Anpassen des 'packaging' Types von 'bundle' in 'jar'
  $ find ./export -iname pom.xml -exec sed -i "s/<packaging>bundle<\/packaging>/<packaging>jar<\/packaging>/g" {} \;

Sourcecode Artefakte fuer jeden exportierten Sourcecode erzeugen.
  mvn -f <POM file name> source:jar
  $ find ./export -mindepth 2 -maxdepth 3 -name pom.xml -exec mvn -f {} clean \;
  $ find ./export -mindepth 2 -maxdepth 3 -name pom.xml -exec mvn -f {} source:jar \;
